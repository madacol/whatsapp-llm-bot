Problem

Media-to-text policy is shallow: callers must know content traversal, quote recursion, current text, context reset, model fallback, and output shape.

Solution

Wins

  • leverage: one test surface
  • audio behavior stops drifting
  • quote recursion fixed once

Problem

Turn Orchestration is shallow because its interface requires harness selection, adapter lifecycle, live input races, runtime event filtering, and durable cursor policy.

Solution

WhatsApp Presentation
OutboundEvent stream to delivery plans
Baileys Delivery Adapter
send, edit, pin, relay media

Problem

WhatsApp outbound is shallow: rendering, status state, edit handles, snapshot continuation, and Baileys delivery share one large interface.

Solution

Move stateful Presentation decisions into a delivery-plan module and keep Baileys calls in a concrete adapter behind the seam.

Wins

  • locality: display policy isolated
  • leverage: transport-free tests
  • OutboundEvent remains canonical
  • delivery adapter gets smaller

Top recommendation

Deepen Media Input Enrichment

Tackle this first because it is narrow, well-tested, and repeats the same media-to-text policy across multiple current seams; it should produce quick locality and leverage without reopening ADRs.
