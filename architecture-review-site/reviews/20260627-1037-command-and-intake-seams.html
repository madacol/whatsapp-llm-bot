Problem

The command path is shallow: Input Orchestration must know bang commands, slash commands, workspace commands, clear follow-ups, persistence timing, Agent Runtime command handling, and AppOutputPort replies.

Solution

Move command parsing, execution policy, persistence, clear follow-ups, and command output creation behind one app-owned Command Orchestration module. Keep ChannelInput routing focused on route policy.

Wins

  • Locality: command bugs concentrate.
  • Leverage: one command test surface.
  • Input Orchestration interface shrinks.
  • Clear flow stops leaking.
  • Output ports stay ownership-specific.

Problem

The HTTP wire module owns repeated turn lifecycle behaviour for text and audio, so idempotency, wait mode, active-turn correlation, failure status, and assistant speech post-processing are not local.

Solution

Put text and audio execution through one deeper HTTP API Transport turn intake module. Leave route matching, auth, CORS, and body parsing in the wire module.

Wins

  • Locality: one lifecycle policy.
  • Leverage: text and audio share tests.
  • Failure handling stops duplicating.
  • Speech post-processing isolates.
  • No speculative app-wide seam.

Chat Settings command asks for settings interaction

Chat Settings module

definition registry

Problem

The settings interface leaks definition internals: callers know picker and multi-picker rules, admin checks, root database wiring, and the difference between inspect, set, and reset flows.

Solution

Move the complete settings interaction behind the Chat Settings module while keeping setting definitions and storage compatibility inside that same implementation.

Wins

  • Locality: settings rules concentrate.
  • Leverage: command stays tiny.
  • Picker tests hit one interface.
  • Definition helpers stop leaking.
  • Output wording stays consistent.

Problem

The app glossary says new app-level interfaces should prefer Channel vocabulary, but many app seams still use chatId for Transport identity, Channel identity, storage identity, and path identity.

Solution

Migrate Channel identity only at new or touched app seams, while adapters and storage keep legacy chat fields at compatibility edges.

Wins

  • Locality: identity translation localizes.
  • Leverage: Transports stop leaking names.
  • Vocabulary migration becomes incremental.
  • Storage compatibility remains intact.
  • Tests can name Channel behavior.

Top Recommendation

Start with Deepen Command Orchestration
