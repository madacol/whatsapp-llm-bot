Problem. The `runtime_event` interface is shallow: callers must understand runtime tool names, arguments, suppression flags, and subagent metadata.

Solution. Deepen Agent Run Activity so it emits semantic OutboundEvents for tool activity, reasoning, subagent activity, and lifecycle state.

  • locality: one event seam
  • leverage: one Presentation input
  • tests hit one interface
  • harness vocabulary shrinks
  • raw facts stop earlier
  • subagent policy centralizes
ADR fit: supports ADR 0001, ADR 0003, and ADR 0004 by replacing runtime pass-through with app-owned OutboundEvent fields.

Problem. Subagent identity is split across the ACP patch, runtime model, dispatcher map, and WhatsApp Presentation.

Solution. Add a Subagent Activity Context module inside Agent Run Activity; the ACP adapter supplies neutral facts and the module owns mapping and fallback naming.

Problem. Thought repetition can hide in multiple modules because the reasoning interface does not own reconciliation as a single behaviour.

Solution. Deepen a Reasoning Trace module that owns coalescing, snapshot replacement, encrypted content, and completion semantics.

  • locality: repeats debug once
  • leverage: provider shapes hidden
  • interface is test surface
  • pinned status simplified
  • encrypted path explicit
  • fixtures become smaller
