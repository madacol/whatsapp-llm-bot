Problem

Creation ownership leaks across the seam; callers learn the full OutboundEvent interface.

Solution

Put semantic creation behind AgentRunOutputPort and AppOutputPort; keep delivery behind OutboundEventSink.

Wins

  • locality: producer rules concentrate
  • leverage: one interface, many callers
  • tests hit owned seams
  • ADR 0004 becomes executable

Move ACP control behind an Agent Runtime command module

Worth exploring local-substitutable

Before

interface: command strings, context, session controls, config state, reply events
implementation: ACP command behavior
ACP adapter emits app output directly

After

interface: semantic runtime command
implementation: parse, validate, persist, apply ACP control, report via AppOutputPort
Agent Runtime command module absorbs provider control details.

Problem

ACP adapter command handling is shallow: its interface exposes provider control, session control, and app output facts together.

Solution

Deepen a command module behind the Agent Runtime seam; Turn Orchestration passes semantic commands only.

Wins

  • locality: ACP control concentrates
  • leverage: one command interface
  • harness vocabulary shrinks
  • tests cross runtime seam

Top recommendation

Deepen OutboundEvent producer ports
